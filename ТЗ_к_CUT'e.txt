1. Введение
   - Описание проекта: Создание мессенджера "CUT'e", включающего клиентскую и серверную части.
   - Макет в Figma = https://www.figma.com/file/dMwJMmZbD5JoQww9ODPB0c/Untitled-(Copy)?node-id=0%3A1&
2. Функциональные требования
   - Регистрация и вход в систему с использованием логина.
   - Отправка и прием текстовых сообщений между пользователями.(WebSocket)
   - Возможность создания групповых чатов.(Зависимость от id пользователя?)
   - Поддержка отправки и просмотра различных медиа-файлов: изображения, видео, аудио.
   - Уведомления о новых сообщениях.
   - Хранение истории переписки в db.
   - Онлайн-статус пользователей.
   - Возможность просмотра профилей пользователей.

3. Требования к клиентской части
   - Использование Vite для сборки проекта.
   - Разработка интерфейса с использованием React.
   - Адаптивный дизайн для поддержки различных устройств.
   - Роутинг страниц.

4. Требования к серверной части
   - Использование JavaScript для серверной части приложения.
   - Использование Express.js для обработки HTTP запросов.
   - Реализация аутентификации и авторизации пользователей.
   - Хранение данных пользователей и сообщений в базе данных (MySql).

5. Дизайн
   - Верстка макета
   - Css стили,Js анимации.

6. Безопасность
   - Обеспечение безопасности передачи данных и хранения пользовательской информации(SSL,хэш).

7. Интеграция
   - Интеграция с внешними API для отправки медиа-файлов (изображения, видео, аудио).
   - API для взаимодействия между клиентской и серверной частями приложения.

10. Задачи
   - Разработка клиентской части приложения с учетом дизайнерского макета.
   - Реализация серверной логики для обмена сообщениями, хранения данных и аутентификации пользователей.
   - Тестирование и отладка приложения.

11. Запуск и поддержка
   - Развертывание приложения на VDS/VPS хостинге.
   - Поддержка и обновление приложения после запуска.
